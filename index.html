RedirectionUtils Manual

Usage: https://[YourDomain]/[Route]?[to|delete]&[password]

to: The actual URL that you want to redirect to.

delete: A flag to delete the existing redirection record. This can only be set to true.

password: The password generated by the server side to authorize the deletion / modification of the redirection record.

Examples:

GET https://[YourDomain]/test?to=https://example.com/

RETURN {code:0,message:"Success",password:"examplepassword1"}

GET https://[YourDomain]/test?to=https://new.example.com/&password=examplepassword1

RETURN {code:0,message:"Success"}

GET https://[YourDomain]/test?password=examplepassword1&delete=true

RETURN {code:0,message:"Success"}

Status Codes:

0 - Success

-1 - Operation was unsuccessful (see message)

-2 - Operation was unsuccessful due to database error

-3 - Insufficient permission / Incorrect password

Secure Deployment

- Use Nginx as a reverse proxy

- Use tools like Gunicorn to run this flask app

- Enable access control to the database
